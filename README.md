CS 330 – Computational Graphics and Visualization Reflection
How do I approach designing software?
On this project I approached design by first breaking the problem into clear components: scene setup, camera/navigation, rendering, and interaction. I sketched out what objects needed to exist (desk, monitor, peripherals, lights) and what responsibilities each class should have (SceneManager, ViewManager, ShaderManager, ShapeMeshes). My design focused on reusing a small set of primitives and functions (for transforms, materials, and textures) to keep the structure simple and extensible. This project helped me practice thinking in terms of data (meshes, materials, lights) plus behavior (update, render, collision) rather than writing everything inline.

What new design skills did this project help me craft?
This work improved my ability to design around the GPU pipeline: I had to think about how vertex data, textures, materials, and lights flow through shaders. I also learned to design helper functions—like functions to set the model matrix, materials, textures, and UV scales—so that high‑level scene code stays readable. Another important skill was thinking spatially: deciding how to place objects in 3D space relative to each other so the scene matches a reference image and feels coherent from multiple camera angles.

What design process did I follow, and how can I reuse it?
I followed an iterative design process: start with a minimal scene (plane + box), then progressively refine by adding more objects, textures, and lighting. Each milestone added one design concern (geometry, navigation, textures, lighting) without trying to solve everything at once. I validated each step visually by running the program and adjusting positions and parameters. This process—start simple, add one concern at a time, verify visually—is reusable for future graphics or UI work, and even for general application design where features are layered gradually.

How do I approach developing programs?
I approach development by separating concerns and encapsulating repeated logic. For this project, I used classes for scene management and viewing, and small reusable functions for common OpenGL tasks. I relied on incremental development: after each small change (adding a new object, tweak to a shader, or a new control), I rebuilt and tested right away. Debugging was often visual (checking object placement, lighting balance), combined with console output for issues like missing textures or shader compile errors.

What new development strategies did I use for the 3D scene, and how did iteration factor in?
New strategies included working directly with GLSL shaders and understanding uniform data flow, as well as using a camera class to manage view and projection. I also became more deliberate about using transformation helpers instead of recomputing matrices manually. Iteration was central: I would first get a bare object on screen, then refine its scale and position, then add texture, then configure lighting and materials. The milestones naturally enforced this iterative cycle, so by the time I reached the final project I had a stable base to extend instead of starting from scratch.

How has my approach evolved through the milestones?
At the start, I focused mostly on “making something appear” on screen. As the milestones progressed, my approach shifted toward designing reusable systems: a single render path that supports any object, a camera that works with both perspective and orthographic projections, and shaders that handle both colored and textured objects with lighting. I became more comfortable modifying and extending existing structures instead of writing ad‑hoc code in main. Overall, my code moved from one‑off solutions toward a more modular, engine‑like structure.

How can computer science help me reach my goals, and how do graphics/visualizations fit in?
Computer science gives me the tools to build interactive systems, simulations, and visual interfaces—skills that are useful whether I pursue game development, data visualization, UI/UX, or other software fields. Computational graphics and visualizations, in particular, add the ability to turn abstract data and ideas into something visual and interactive. Educationally, this opens the door to more advanced courses in graphics, game programming, or simulation. Professionally, understanding the rendering pipeline, transformations, and basic physics/interaction makes me more prepared for roles that involve 3D engines, visualization dashboards, or any application where visual feedback and user interaction are important.
